# Git 동기화 시작과 끝내기 가이드

이 문서는 작업을 시작할 때의 코드 가져오기(Pull)와 작업을 끝마칠 때의 코드 올리기(Commit & Push) 절차를 정리한 퀵 가이드입니다. 

터미널이나 에이전트 채팅창에서 동일한 습관으로 사용할 수 있습니다.

---

## 📥 1. 작업 시작할 때 (불러오기)
다른 PC나 작업자가 수정한 최신 코드를 내 로컬 컴퓨터로 가져옵니다.

*   **실행할 명령어**:
    ```powershell
    git pull origin main
    ```
*   **에이전트 요청 예시**: 
    > "git pull origin main 해줘"

---

## 📤 2. 작업 끝마칠 때 (저장 및 올리기)
내가 작업하여 수정한 코드를 로컬 저장소에 기록하고, GitHub 원격 저장소에 업로드합니다.

*   **실행할 명령어 (순서대로)**:
    1.  **스테이징**: 수정된 전체 파일을 선택합니다.
        ```powershell
        git add .
        ```
    2.  **커밋 생성**: 변경 내용에 대한 메모(메시지)를 남깁니다.
        ```powershell
        git commit -m "작업 내용 설명 작성"
        ```
    3.  **원격 전송 (Push)**: GitHub로 업로드합니다.
        ```powershell
        git push origin main
        ```

*   **에이전트 요청 예시**:
    > "git add . 하고 git commit -m '작업 완료' 한 다음 git push origin main 해줘"

---

## 💡 요약 표

| 작업 단계 | 실행 명령어 | 에이전트 요청 한마디 |
| :--- | :--- | :--- |
| **시작 (불러오기)** | `git pull origin main` | "git pull 해줘" |
| **끝내기 (올리기)** | `git add .`<br>`git commit -m "설명"`<br>`git push origin main` | "add, commit, push 일괄 처리해줘" |
